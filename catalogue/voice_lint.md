# voice_lint -- rule rendering

This page is the public rendering of the rules enforced by the
`voice_lint` gate. The canonical source is the python constants in
`scripts/lib/voice_lint.py`; this file is regenerated by hand when the
constants change. `voice_lint` itself skips the `catalogue/` directory
when scanning a host repo (see `decisions/DEC-001-gate-rule-corpus-v0.md`),
so listing the words here does not flag the gate against its own
catalog.

## scope

- file extensions scanned: `.md`, `.txt`, `.rst`
- directories skipped by default: `.git`, `.venv`, `venv`,
  `node_modules`, `__pycache__`, `.pytest_cache`, `specs`, `decisions`,
  `catalogue`, `examples`
- a host may point `--path` directly at one of the skipped directories
  to force a scan

## rule: banlist

each banned word produces a `fail` finding with
`rule_id = voice_lint::banlist::<word>`. match is case-insensitive and
word-bounded. the banlist:

| word | rationale |
|---|---|
| leverage | marketing verb; prefer "use" |
| synergy | marketing noun with no concrete referent |
| robust | unsupported quality claim |
| demonstrates | telling not showing; prefer the demonstration itself |
| comprehensive | unsupported scope claim |
| best-in-class | unsupported comparison claim |
| unleash | marketing verb |

## rule: antithetical reversal

sentence shapes that signal a structural AI tell. one `fail` finding
per matched line, `rule_id = voice_lint::reversal`:

- `not X but Y` (within a line)
- `it's not X; it's Y` (single line, semicolon- or colon-separated)

## known limits in v0

- code fences are not skipped; a banned word inside a fenced block is
  still flagged. fence-aware skipping is deferred to spec 0003.
- the reversal patterns only match within a single line.
